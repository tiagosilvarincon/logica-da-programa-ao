uma branch funciona como uma cópia do código principal de um projeto. 
Ela permite que os desenvolvedores façam alterações sem mexer diretamente na versão principal. 
Isso ajuda a evitar problemas no sistema enquanto novas mudanças estão sendo feitas. 
As branches são muito importantes para manter a organização do projeto e tornam o trabalho mais seguro para toda a equipe.
 Quando uma alteração ainda não está pronta, ela pode ficar isolada em uma branch. 
Assim, os usuários não são afetados por erros ou funções incompletas. 
Depois que tudo é testado e aprovado, as mudanças podem ser integradas ao código principal. 
Um exemplo é a correção de um bug encontrado no sistema.
 O desenvolvedor cria uma branch específica para resolver esse problema. 
Dessa forma, o código principal continua funcionando normalmente. 
Após os testes, a correção é adicionada ao projeto.
Outro exemplo é a criação de uma nova funcionalidade. 
Uma opção de recuperação de senha pode ser desenvolvida em uma branch separada. 
Isso permite trabalhar com mais alivi com uma tranquilidade e uma segurança muito boa
para seu trebalho.
